# Feature list 
Produce a Feature list on the format
For features: <action> the <result> <by|for|of|to> a(n) <object>
For feature sets: <action><-ing> a(n) <object> where an object is a person, place, or thing (including roles, moments in time or intervals of time, or catalog-entry-like descriptions) That addresses the requirements.

R1 Read data
Feature Set: <Reading> a <temperature data file> 
Feature: <Load> the <data from a temperature file>
Feature: <Parse> the <time and temperature> in <a given input file>
Feature: <Display> a <message> for <successful/failed data read-in>
Feuture: <Report> an <error> for <missing or corrupted temperature entries>
